# Contributing to Spirelink

Spirelink is a community project. The most valuable contributions are **data updates** — keeping card text, synergy edges, and rarity information accurate as Slay the Spire 2 evolves through Early Access.

You do not need to know how to code to contribute. If you know StS2 card mechanics, you can contribute.

---

## Project structure

```
data/
  shared/
    relationship-types.json   ← edge type definitions (rarely changes)
  defect/
    nodes.json                ← all cards, relics, and mechanics for Defect
    edges.json                ← synergy relationships between nodes
src/                          ← React application (touch only if adding features)
```

---

## Adding or updating a card (nodes.json)

Each node looks like this:

```json
{
  "id": "ice_lance",
  "label": "Ice Lance",
  "type": "card",
  "rarity": "uncommon",
  "cost": 2,
  "desc": "Deal 19 damage. Channel 3 Frost. Powerful all-in-one card.",
  "archetype": ["wall_of_ice"]
}
```

### Field reference

| Field | Values | Notes |
|-------|--------|-------|
| `id` | snake_case string | Must be unique. Used as the key everywhere. |
| `label` | Display string | The name shown on the graph. |
| `type` | `"card"` `"relic"` `"mechanic"` | Mechanics are abstract (Focus, Orb Slots, etc.) |
| `rarity` | `"starter"` `"common"` `"uncommon"` `"rare"` `"relic"` `null` | `null` for mechanics |
| `cost` | integer or omit | Card energy cost. Omit for relics and mechanics. |
| `desc` | string | One or two sentences. What does it do, and why does it matter? |
| `archetype` | array of strings | Which build archetypes this card belongs to. See below. |

### Archetypes (Defect)

- `wall_of_ice` — Frost orb stacking + passive block
- `lord_of_darkness` — Dark orb scaling + Evoke burst
- `pure_lightning` — Lightning orb AoE damage

## Adding a synergy edge (edges.json)

Each edge looks like this:

```json
{
  "from": "glacier",
  "to": "frost_orb",
  "type": "generates",
  "desc": "Channels 2 Frost while gaining Block — exceptional double value per energy spent."
}
```

### Field reference

| Field | Values | Notes |
|-------|--------|-------|
| `from` | node `id` | The source of the relationship. |
| `to` | node `id` | The target of the relationship. |
| `type` | see below | The nature of the synergy. |
| `desc` | string | Explain *why* this relationship matters, not just that it exists. |

### Edge types

| Type | Meaning |
|------|---------|
| `generates` | The source card/relic produces the target resource or mechanic |
| `amplifies` | The source increases the power or value of the target |
| `triggers` | The source activates or conditions the target mechanic |
| `scales_with` | The source becomes more powerful as the target resource grows |
| `conflict` | The source works against or undermines the target — **anti-synergy** |

---

## What makes a good description?

Bad: *"Ice Lance works with Hailstorm."*

Good: *"Each Frost generated by Ice Lance feeds Hailstorm's end-of-turn AoE condition — the more you channel, the more reliably Hailstorm fires."*

The description should teach the player *why* the synergy exists, not just that it does.

## Adding a new character

1. Create `data/<character>/nodes.json` and `data/<character>/edges.json` following the same schema.
2. Update `src/App.jsx` to load the new character's data (or open an issue and let a developer handle the wiring).

---

## Reporting outdated data

If a card's cost, effect, or rarity has changed in an EA patch, open an issue with:
- The card name
- What changed
- The patch version if known

EA moves fast — stale data is expected and not a criticism of prior contributors.

---

## Code contributions

Standard GitHub flow: fork → branch → PR. Please keep PRs focused — one feature or fix per PR.

For significant feature changes, open an issue first to discuss direction before building.
